Capability surface

Built around Redis source semantics

Redis does not expose one universal CDC stream. The gem keeps each source mode explicit so replay, loss, checkpointing, and recovery guarantees stay visible to operators.

Redis Streams

XREAD, XREADGROUP, consumer groups, checkpoint inspection, pending-entry recovery, XAUTOCLAIM, duplicate suppression, and optional dead-letter streams.

Ephemeral channels

Exact Pub/Sub, pattern Pub/Sub, and Redis Cluster sharded Pub/Sub with readiness tracking, reconnect metrics, and explicit loss-window reporting.

Keyspace notifications

Startup validation for notify-keyspace-events, optional best-effort value enrichment, and Cluster multi-primary subscriptions through CLUSTER SHARDS.

Delivery model

Guarantees are source-specific

Source Delivery Replay Operational note
Streams At-least-once Yes Acknowledges and advances checkpoints only after successful processing.
Pub/Sub At-most-once No Reconnects report loss windows; missed messages are not recovered.
Sharded Pub/Sub At-most-once No Routes channels by slot and rebinds subscriptions after topology changes.
Keyspace notifications At-most-once No Reports key and operation; optional enrichment is best effort.

Production topology

Sentinel, Cluster, TLS, and ACLs

Connection setup is topology-neutral at the source API boundary. Source-owned clients reconnect through the same factory, while Redis Cluster commands route through redis-cluster-client.

Integration shape

Fast Redis source to bounded runner

The private gem exposes Redis-native source events, a cdc-core adapter, and a Phase 6 SourceRunner for bounded processing, lifecycle health, and orchestrator-friendly error observation. Redis is positioned as the fast upstream signal layer; PostgreSQL remains the stronger durability baseline. 

require "cdc/redis/pro"

source = CDC::Redis::Pro::Streams::Source.new(
  connection: { url: ENV.fetch("REDIS_URL") },
  streams: ["orders"],
  group: "cdc",
  consumer: "worker-1",
  checkpoint_store: CDC::Redis::Pro::MemoryCheckpointStore.new,
  dead_letter_stream: "orders.dead"
)

adapter = CDC::Redis::Pro::SourceAdapter.new

runner = CDC::Redis::Pro::SourceRunner.new(
  source: source,
  adapter: adapter,
  queue_size: 16,
  worker_count: 2
)

runner.start do |change_event, source_event|
  process(change_event)
end

Phase 6 proof layer

Redis feed, cdc-orchestrator-pro downstream

Phase 6 showcases the pairing: a very fast Redis upstream source feeding cdc-orchestrator-pro downstream primitives. The public benchmark harness exercises HybridRuntime directly with cdc-parallel Ractors, cdc-concurrent fiber fan-out, and Ractor-local Redis connection pools.

Inception pool

The runtime shape is source to CDC::Core::ChangeEvent, partition router, prewarmed Ractor pool, Ractor-local Redis pool, and concurrent fiber fan-out. This keeps the less durable Redis signal separated from the heavier downstream execution layer.

Benchmark matrix

The public harness varies 1, 3, and 7 Ractors; 1, 5, and 10 Redis connections per Ractor; 10, 50, and 100 fibers; and ordered versus unordered delivery. It reports throughput, latency, CPU, RSS, Redis ops, mailbox depth, and failures.

Observed run

The latest pgoutput-to-Redis matrix completed 54 of 54 stable runs with a peak of 19,977 events/s. A Streams soak processed 1.34M events over five minutes with 0 failures and 2.1% median throughput degradation.

Redis Streams, Pub/Sub, and keyspace notifications are intentionally presented with different replay and loss semantics. The benchmark demonstrates orchestration capacity; it does not claim Redis Pub/Sub or keyspace notifications are as durable as PostgreSQL WAL. 
sink = CDC::Redis::Pro::Orchestrator.redis_sink(
  connection: { url: ENV.fetch("REDIS_URL") },
  redis_connections: 5,
  key_prefix: "orders_projection",
  id_field: "stream_id",
  value_field: "payload"
)

runtime = CDC::Redis::Pro::Orchestrator.runtime(
  processor: sink,
  ractors: 3,
  fiber_concurrency: 50,
  timeout: 30.0,
  preserve_order: true,
  partition_strategy: :round_robin
)

results = runtime.process_many(change_events)
bundle exec cdc-redis-pro orchestrate \
